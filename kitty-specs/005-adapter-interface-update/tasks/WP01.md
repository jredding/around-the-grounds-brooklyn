# WP01 — Update BaseParser type signatures

## Goal
Replace all `Brewery` and `FoodTruckEvent` references in `base.py` with `Venue` and `Event` across imports, method signatures, and field access — no logic change.

## Context
- Upstream-first: clean PR to around-the-grounds main
- Minimalism: only type signatures and field names change — no logic, no structure
- Prerequisite: spec 004 merged to main before implementing (Venue and Event must be importable)
- Branch: `005-adapter-interface-update`

## Subtasks
- [ ] Import L8: `from ..models import Brewery, FoodTruckEvent` → `from ..models import Venue, Event`
- [ ] `__init__` sig: `def __init__(self, brewery: Brewery):` → `def __init__(self, venue: Venue):`
- [ ] `__init__` body: `self.brewery = brewery` → `self.venue = venue`
- [ ] `parse()` return type: `List[FoodTruckEvent]` → `List[Event]`
- [ ] `validate_event()` sig: `event: FoodTruckEvent` → `event: Event`
- [ ] `validate_event()` body: `event.brewery_key` → `event.venue_key`
- [ ] `validate_event()` body: `event.brewery_name` → `event.venue_name`
- [ ] `validate_event()` body: `event.food_truck_name` → `event.title`
- [ ] `validate_event()` body: `event.date` → `event.datetime_start`
- [ ] `filter_valid_events()` sig: `List[FoodTruckEvent]` → `List[Event]` (both param and return)

## Files to change
- `around_the_grounds/parsers/base.py` — 10 atomic edits, no other files

## Acceptance criteria
- `python -c "from around_the_grounds.parsers.base import BaseParser"` — no import error
- `parsers/registry.py` diff is empty (confirm unchanged)
- No logic change of any kind; only type signatures and field name references updated

## Reviewer notes
This WP is a prerequisite for WP02. All 7 parser files inherit from `BaseParser`, so the
`__init__` parameter rename (`brewery` → `venue`) and `self.venue` assignment must land
